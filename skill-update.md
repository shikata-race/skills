# skill-update — スキルに学びを書き込む

作業後の気づき・コツ・パターンをスキルファイルの学習ログに追記してGitHubにpushする。
**プロジェクト固有の情報（ID・会社名・ファイル名）は書かず、次に別のプロジェクトでも使える知識だけを残す。**

## 使い方

```
/skill-update [skill-name]
```

引数省略時はどのスキルを更新するかユーザーに確認する。

---

## 実行手順

### STEP 1: 何を記録するか確認

ユーザーに以下を質問する：

**「どのスキルに記録しますか？また、今回の作業で気づいた『次回も使えるコツや注意点』を教えてください」**

記録すべき情報の例：
- ✅ 「〇〇の順番でやると効率よかった」
- ✅ 「〇〇パターンはコンフリクトしやすいので△△で回避する」
- ✅ 「エージェントへの指示で〇〇を明示すると精度が上がった」
- ❌ プロジェクト名、会社名、ファイルの具体的なID
- ❌ 「今回のXXX機能では〜」（プロジェクト文脈に依存した記述）

### STEP 2: 対象スキルファイルを読む

```
~/.claude/skills-repo/<skill-name>.md
```

`## 学習ログ` セクションの位置と既存エントリを確認する。

### STEP 3: 学習ログに追記

`## 学習ログ` セクションに追記（新しいエントリを末尾に）：

```markdown
### YYYY-MM-DD
- **効いたパターン**: ...（再現性のある手法）
- **注意**: ...（はまったこと・回避策）
- **コツ**: ...（小さな工夫）
```

フォーマットルール：
- 箇条書き。1行で完結する粒度にする
- 「なぜ効くか」の理由も一言添えると再利用しやすい
- プロジェクト名・会社名・具体的なID等は含めない

Edit ツールで `~/.claude/skills-repo/<skill-name>.md` を更新する。

### STEP 4: REGISTRY.md の最終更新日を更新

```
~/.claude/skills-repo/REGISTRY.md
```

対象スキルの「最終更新」列を今日の日付に変更する。

### STEP 5: GitHubにpush

```bash
cd ~/.claude/skills-repo
git add <skill-name>.md REGISTRY.md
git commit -m "learn(<skill-name>): YYYY-MM-DD の学習ログ追加"
git push origin main
```

### STEP 6: ~/.claude/commands/ にも反映

```bash
cp ~/.claude/skills-repo/<skill-name>.md ~/.claude/commands/<skill-name>.md
```

---

## スキルの手順・ルール自体を改善したい場合

学習ログへの追記ではなく、スキルの本文を書き直す場合：

1. STEP 2でファイルを読む
2. 改善箇所を Edit で修正
3. コミットメッセージは `fix:` か `feat:` を使う
